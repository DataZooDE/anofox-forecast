# AutoMSTL

> AutoMSTL with automatic model selection for multi-seasonality

## Signature

```sql
-- Single series
SELECT * FROM ts_forecast('table', date_col, value_col, 'AutoMSTL', horizon, params);

-- Multiple series (grouped)
SELECT * FROM ts_forecast_by('table', group_col, date_col, value_col, 'AutoMSTL', horizon, params);
```

## Description

Multiple Seasonal-Trend decomposition using LOESS with automatic model selection. Decomposes the series into trend, multiple seasonal components, and remainder. Forecasts are generated by extrapolating each component.

> **Important:** Seasonality is NOT auto-detected. Use `ts_detect_periods_by` first to detect seasonal periods, then pass them explicitly via `seasonal_periods`.

## Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `horizon` | INTEGER | Yes | — | Number of periods to forecast |
| `seasonal_periods` | INTEGER[] | **Yes** | — | Array of seasonal periods (e.g., `'[7, 365]'` for weekly+yearly) |
| `confidence_level` | DOUBLE | No | 0.95 | Confidence for prediction intervals |
| `include_fitted` | BOOLEAN | No | false | Return in-sample fitted values |
| `include_residuals` | BOOLEAN | No | false | Return residuals |

## Returns

| Column | Type | Description |
|--------|------|-------------|
| `group_col` | ANY | Series identifier (only for `_by` variant) |
| `ds` | TIMESTAMP | Forecast timestamp |
| `forecast` | DOUBLE | Point forecast |
| `lower` | DOUBLE | Lower prediction interval |
| `upper` | DOUBLE | Upper prediction interval |

## SQL Example

```sql
-- Step 1: Detect seasonality
SELECT * FROM ts_detect_periods_by('daily_sales', product_id, date, quantity, MAP{});
-- Returns: periods = [7, 365] (weekly and yearly)

-- Step 2: Forecast with detected seasonal periods
SELECT * FROM ts_forecast_by(
    'daily_sales',
    product_id,
    date,
    quantity,
    'AutoMSTL',
    90,
    {'seasonal_periods': '[7, 365]'}
);
```

## Best For

- Data with multiple seasonal patterns at different frequencies
- Long time series with yearly and weekly seasonality
- When you need interpretable decomposition
- Retail, energy, and traffic data with complex patterns
