# AutoMSTL

## Signature

```sql
-- Single series
SELECT * FROM ts_forecast('table', date_col, value_col, 'AutoMSTL', horizon, params);

-- Multiple series (grouped)
SELECT * FROM ts_forecast_by('table', group_col, date_col, value_col, 'AutoMSTL', horizon, params);
```

## Description

Automatic Multiple Seasonal-Trend decomposition using LOESS. Automatically detects multiple seasonal patterns and decomposes the series into trend, multiple seasonal components, and remainder. Forecasts are generated by extrapolating each component.

## Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `horizon` | INTEGER | Yes | â€” | Number of periods to forecast |
| `seasonal_periods` | INTEGER[] | No | auto-detect | Array of seasonal periods |
| `confidence_level` | DOUBLE | No | 0.95 | Confidence for prediction intervals |
| `include_fitted` | BOOLEAN | No | false | Return in-sample fitted values |
| `include_residuals` | BOOLEAN | No | false | Return residuals |

## Returns

| Column | Type | Description |
|--------|------|-------------|
| `group_col` | ANY | Series identifier (only for `_by` variant) |
| `ds` | TIMESTAMP | Forecast timestamp |
| `forecast` | DOUBLE | Point forecast |
| `lower` | DOUBLE | Lower prediction interval |
| `upper` | DOUBLE | Upper prediction interval |

## SQL Example

```sql
-- Basic usage (auto-detect seasonality)
SELECT * FROM ts_forecast_by(
    'daily_revenue',
    store_id,
    date,
    revenue,
    'AutoMSTL',
    30,
    {}
);

-- With explicit multiple seasonal periods (weekly and yearly)
SELECT * FROM ts_forecast_by(
    'daily_sales',
    product_id,
    date,
    quantity,
    'AutoMSTL',
    90,
    {'seasonal_periods': '[7, 365]'}
);
```

## Best For

- Data with multiple seasonal patterns at different frequencies
- Long time series with yearly and weekly seasonality
- When you need interpretable decomposition
- Retail, energy, and traffic data with complex patterns
